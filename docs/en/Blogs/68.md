# $mq9.AI.* Protocol Design

## Overview

`$mq9.AI.*` is mq9's protocol for Agent asynchronous communication. The core problem solved: **Asynchronous communication between Agents — sender and receiver do not need to be online simultaneously.**

mq9 only solves the communication problem — how to reliably deliver messages. What the message contains, how it is used, and what the business semantics are — mq9 does not care. Message content is a byte array: not parsed, not validated, not restricted.

---

## mail_address Format Specification

**Format**: `mq9://host[:port]/{mail_address}` or `mq9s://host[:port]/{mail_address}`

**Character set**: Lowercase letters (a-z), digits (0-9), and dots (`.`)

**Length**: 1 to 128 characters

**Case**: Strictly lowercase — mail_addresses containing uppercase letters are rejected by the broker

**Position rules**: `.` may only appear in the middle; the first and last characters must be a lowercase letter or digit; consecutive dots are not allowed

**Semantics**: `mail_address` is an opaque string — `.` has no routing or matching significance and serves only as a visual separator

**Encoding**: URL percent-encoding is not allowed

| Valid examples | Invalid examples |
|---------------|-----------------|
| `task.001` | `task-001` (hyphen) |
| `agent.inbox` | `task_001` (underscore) |
| `analytics.result` | `Task.001` (uppercase) |
| `acme.org.task.queue` | `.task.001` (leading dot) |
| `session.20260502` | `task.001.` (trailing dot) |
| | `task..001` (consecutive dots) |

**Full address examples**:

```text
mq9://127.0.0.1:4332/task.001
mq9://broker.example.com/agent.inbox
mq9s://broker.example.com:4332/analytics.result
mq9s://mq9.acme.org/acme.task.queue
mq9://127.0.0.1:4332/session.20260502
mq9s://broker.example.com/order.processing.urgent
mq9s://broker-east-1.acme.org:9332/agent.001.inbox
mq9://localhost:4332/test.mailbox
```

## Core Concepts

**mail_address**: The communication address returned when creating a mailbox via `MAILBOX.CREATE`. Not bound to Agent identity — one Agent can apply for different mail_addresses for different tasks. Leave it alone when done; TTL handles cleanup automatically.

Private mailbox addresses are system-generated in the format `{uuid}@mq9`. With a prefix, the format is `{prefix}@mq9`. Public mailbox addresses are user-defined with meaningful names — `task.queue@mq9`, `analytics.result@mq9`.

**Unguessability is the security boundary.** Knowing the mail_address lets you send messages and subscribe. Without it, you can't interact with the mailbox. No token, no ACL.

**TTL**: Declared at mailbox creation time; auto-destroyed on expiry along with all messages. CREATE is idempotent — creating the same mailbox again does not error; TTL is fixed by the first creation.

**priority**: Optional. Defaults to `normal`. `urgent` and `critical` raise processing order. Same priority is FIFO; across priorities, higher priority is processed first (`critical > urgent > normal`). The storage layer guarantees ordering; consumers need not sort themselves.

**msg_id**: Each message's unique identifier (its offset in storage). Used by clients for deduplication and deletion.

**Subscription semantics**: Every subscription delivers all non-expired messages in full, in priority order, with new messages pushed in real time afterward. No read/unread distinction, no consume-offset tracking. Zero consumer state on the server.

**Message flow**: Message arrives → written to storage → pushed to online subscribers in real time; if offline, waits → delivered in full on next subscription.

## Protocol Summary

| Subject | Direction | Persisted | Description |
|---------|-----------|-----------|-------------|
| `$mq9.AI.MAILBOX.CREATE` | REQ/REP | — | Create a mailbox |
| `$mq9.AI.MAILBOX.MSG.{mail_address}` | REQ/REP | Yes | Send a message (normal priority) |
| `$mq9.AI.MAILBOX.MSG.{mail_address}.urgent` | REQ/REP | Yes | Send a message (urgent) |
| `$mq9.AI.MAILBOX.MSG.{mail_address}.critical` | REQ/REP | Yes | Send a message (highest priority) |
| `$mq9.AI.MAILBOX.MSG.{mail_address}` | SUB | — | Subscribe to receive messages |
| `$mq9.AI.MAILBOX.LIST.{mail_address}` | REQ/REP | — | List messages in a mailbox |
| `$mq9.AI.MAILBOX.DELETE.{mail_address}.{msg_id}` | REQ/REP | — | Delete a specific message |

> All commands use request/reply mode (`nats request`); the server always returns a response. Subscriptions (SUB) use `nats subscribe`; the server pushes messages proactively.

---

## Unified Response Format

All commands share a single flat JSON response (`Mq9Reply`). Fields are populated on demand:

```json
{
  "error": "",
  "mail_address": "...",
  "is_new": true,
  "msg_id": 8,
  "messages": [...],
  "deleted": true
}
```

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | Always present. Empty on success; error message on failure |
| `mail_address` | string? | Returned by CREATE and LIST on success |
| `is_new` | bool? | Returned by CREATE; `true` = newly created, `false` = already existed |
| `msg_id` | u64? | Returned by MSG on success; the message's storage offset |
| `messages` | array? | Returned by LIST on success; message metadata list |
| `deleted` | bool? | Returned by DELETE on success; always `true` |

## $mq9.AI.MAILBOX.CREATE

Create a mailbox. Private mailbox addresses are system-generated; public mailbox addresses are user-defined.

### CREATE Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `ttl` | u64? | No | Mailbox lifetime in seconds; never expires if omitted |
| `public` | bool | No | Whether the mailbox is public; defaults to `false` |
| `name` | string? | No | Custom name for public mailboxes (only valid when `public: true`) |
| `prefix` | string? | No | Namespace prefix for private mailboxes; only lowercase letters, digits, and dots allowed (e.g. `risk.critical`). Generated address: `{prefix}@mq9` |
| `desc` | string | No | Description |

### CREATE Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | Empty on success; error message on failure |
| `mail_address` | string | The mailbox address |
| `is_new` | bool | `true` = newly created; `false` = already existed (idempotent) |

### CREATE Examples

```bash
# Create a private mailbox
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl": 3600}'
# Response
{"error":"","mail_address":"d7ms26al@mq9","is_new":true}

# Create a private mailbox with a prefix
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl": 3600, "prefix": "risk.critical"}'
# Response
{"error":"","mail_address":"risk.critical@mq9","is_new":true}

# Create a public mailbox
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl":3600,"public":true,"name":"task.queue@mq9","desc":"Task queue"}'
# Response
{"error":"","mail_address":"task.queue@mq9","is_new":true}

# Idempotent — already exists
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl": 3600, "public": true, "name": "task.queue@mq9"}'
# Response
{"error":"","mail_address":"task.queue@mq9","is_new":false}
```

## $mq9.AI.MAILBOX.MSG.{mail_address}

The message channel. Same mail_address: REQ to send, SUB to receive.

### Sending Messages (REQ/REP)

Knowing the mail_address is all that's needed — no authorization required.

#### MSG Request Parameters

| Parameter | Description |
|-----------|-------------|
| Subject | `$mq9.AI.MAILBOX.MSG.{mail_address}` — normal priority (default) |
| Subject | `$mq9.AI.MAILBOX.MSG.{mail_address}.urgent` — urgent |
| Subject | `$mq9.AI.MAILBOX.MSG.{mail_address}.critical` — highest priority |
| Payload | Any byte array; mq9 does not parse it |

#### MSG Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | Empty on success; error message on failure (e.g. mailbox not found) |
| `msg_id` | u64 | The message's storage offset; use for deduplication or deletion |

#### Priority Levels

| Level | Subject Suffix | Semantics | Typical Use |
|-------|---------------|-----------|-------------|
| normal | _(none)_ | Default | Task dispatch, result delivery, status reporting |
| urgent | `.urgent` | Time-sensitive | Approval requests, important notifications |
| critical | `.critical` | Highest priority | Task abort, emergency commands |

Same priority: FIFO. Across priorities: `critical > urgent > normal`.

#### MSG Examples

```bash
nats request '$mq9.AI.MAILBOX.MSG.d7ms26al@mq9' 'hello'
# Response
{"error":"","msg_id":0}

nats request '$mq9.AI.MAILBOX.MSG.d7ms26al@mq9.urgent' 'urgent payload'
# Response
{"error":"","msg_id":1}

nats request '$mq9.AI.MAILBOX.MSG.d7ms26al@mq9.critical' 'abort'
# Response
{"error":"","msg_id":2}

# Mailbox does not exist
nats request '$mq9.AI.MAILBOX.MSG.nonexistent' 'hello'
# Response
{"error":"mailbox nonexistent does not exist"}
```

### Subscribing to a Mailbox (SUB)

```bash
# Subscribe to all messages (delivered in priority order: critical → urgent → normal)
nats subscribe '$mq9.AI.MAILBOX.MSG.{mail_address}'

# Competing consumers (each message handled by exactly one subscriber)
nats subscribe '$mq9.AI.MAILBOX.MSG.{mail_address}' --queue workers
```

Subscribing delivers all non-expired messages immediately in priority order, then pushes new messages in real time.

**Prohibited operations:**

```bash
nats subscribe '$mq9.AI.MAILBOX.MSG.*'  # Rejected by broker
nats subscribe '$mq9.AI.MAILBOX.MSG.#'  # Rejected by broker
```

Subscriptions must specify an exact mail_address. Wildcard subscriptions across all mailboxes are not permitted.

## $mq9.AI.MAILBOX.LIST.{mail_address}

List all message metadata currently stored in a mailbox. Does not affect subscription delivery and does not consume messages.

### LIST Request Parameters

| Parameter | Description |
|-----------|-------------|
| Subject | `$mq9.AI.MAILBOX.LIST.{mail_address}` |
| Payload | Empty (`''`) |

### LIST Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | Empty on success; error message on failure |
| `mail_address` | string | The mailbox address |
| `messages` | array | Message metadata list, sorted by priority |

**`messages` array item fields:**

| Field | Type | Description |
|-------|------|-------------|
| `msg_id` | u64 | Message offset; globally unique; used for deletion |
| `payload` | string | Message content (byte array as string) |
| `priority` | string | `critical` / `urgent` / `normal` |
| `header` | bytes? | NATS Header (optional) |
| `create_time` | u64 | Write timestamp (Unix seconds) |

### LIST Example

```bash
nats request '$mq9.AI.MAILBOX.LIST.d7ms26al@mq9' ''
# Response
{
  "error": "",
  "mail_address": "d7ms26al@mq9",
  "messages": [
    {"msg_id": 2, "payload": "abort",          "priority": "critical", "header": null, "create_time": 1234567892},
    {"msg_id": 1, "payload": "urgent payload", "priority": "urgent",   "header": null, "create_time": 1234567891},
    {"msg_id": 0, "payload": "hello",          "priority": "normal",   "header": null, "create_time": 1234567890}
  ]
}
```

## $mq9.AI.MAILBOX.DELETE.{mail_address}.{msg_id}

Delete a specific message from a mailbox. Processed messages can be explicitly cleaned up to keep the mailbox tidy.

### DELETE Request Parameters

| Parameter | Description |
|-----------|-------------|
| Subject | `$mq9.AI.MAILBOX.DELETE.{mail_address}.{msg_id}` |
| Payload | Empty (`''`) |

`msg_id` is the offset returned by LIST or by the MSG send response.

### DELETE Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | Empty on success; error message on failure |
| `deleted` | bool | Always `true` on success |

### DELETE Example

```bash
nats request '$mq9.AI.MAILBOX.DELETE.d7ms26al@mq9.2' ''
# Response
{"error":"","deleted":true}
```

## Public Mailboxes

Public mailboxes have no separate discovery mechanism. The mail_address is user-defined as a meaningful name — `task.queue@mq9`, `analytics.result@mq9` — anyone who knows the name can subscribe directly:

```bash
nats subscribe '$mq9.AI.MAILBOX.MSG.task.queue@mq9'
```

Create with `public: true` and a custom name:

```bash
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl":3600,"public":true,"name":"task.queue@mq9","desc":"Task queue"}'
# Response
{"error":"","mail_address":"task.queue@mq9","is_new":true}
```

Public and private mailboxes differ only in whether the mail_address is predictable. The communication mechanism is identical.

## Error Reference

| Scenario | Response Example |
|----------|-----------------|
| Mailbox not found (MSG / SUB / LIST / DELETE) | `{"error":"mailbox xxx does not exist"}` |
| Mailbox already exists (CREATE — idempotent, no error) | `{"error":"","mail_address":"xxx","is_new":false}` |
| msg_id not found (DELETE) | `{"error":"message not found"}` |
