# mq9：是什么

## 一句话

mq9 是 RobustMQ 专门为 AI Agent 设计的通信协议层，与 MQTT、Kafka、NATS、AMQP 并列，给 Agent 提供邮箱和广播能力。

![img](../../images/69-1.jpg)
---

## 解决什么问题

Agent 需要通信。但 Agent 不是服务——它是临时的，随时上下线，生命周期可能只有几秒。

今天 Agent A 给 Agent B 发消息，B 不在线，消息直接丢了。每个团队都在用 Redis、轮询数据库、自研队列绕过这个问题。

mq9 直接解决它：**发出去，对方上线自然收到。**

---

## 三个原语

**邮箱**：每个 Agent 有自己的邮箱，点对点异步通信，对方不在线消息等着。

**广播**：一发多收，谁关心谁订阅，发布方不需要知道谁在听。

**优先级**：紧急消息先处理，不被普通任务淹没。

```bash
# 一行命令，申请邮箱
nats req '$mq9.AI.MAILBOX.CREATE' '{}'

# 发消息
nats pub '$mq9.AI.INBOX.{agent_id}.normal' '{"from":"A","payload":"..."}'

# 广播
nats pub '$mq9.AI.BROADCAST.task.available' '{"task_id":"t-001"}'
```

---

## 为什么不需要新 SDK

mq9 基于 NATS 协议。所有语言的 NATS 客户端——Go、Python、Rust、Java——直接就是 mq9 的客户端。不需要安装任何新东西。

---

## 与 RobustMQ 的关系
mq9 是 RobustMQ 的第五个原生协议，与 MQTT、Kafka、NATS、AMQP 共享同一套统一存储架构。部署一个 RobustMQ，mq9 的能力全部就位。
