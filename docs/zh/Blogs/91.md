# 找个人一起走

我想找一个人。最多两个。

对 AI Infra 有真实兴趣，对 Agent 生态、Agent 通信、下一代 AI 基础设施会长什么样有强烈探索欲的人。一起做 mq9。

---

## 我们在做什么

mq9 是一个为 AI Agent 设计的异步通信协议。

Agent 不是人类应用。它有身份、有状态、会休眠、会离线、需要和其他 Agent 跨时间协作。这套行为模式，Kafka 太重，HTTP 太同步，Redis Pub/Sub 不持久化，NATS 是好底座但缺 Agent 原生的抽象。

mq9 想填这一层。每个 Agent 一个邮箱，消息投递到邮箱，Agent 按自己的节奏处理。

协议设计成型了，核心功能跑通了，LangChain 集成发了，A2A 的协作模式理清了。官网 [mq9.robustmq.com](https://mq9.robustmq.com/)，GitHub [robustmq/mq9](https://github.com/robustmq/mq9)。

但事情太多了，搞不过来。

## 想做的事

下面这些事，每一件都想做，每一件都拿不出够的时间：

1. **mq9 Agent-to-Agent 异步通信内核开发**
2. **多语言 SDK**（Python / Go / Rust / TypeScript / Java）
3. **LangChain & LangGraph 深度集成**
4. **OpenClaw mq9 plugin** —— 让 24 万 star 的 OpenClaw 社区用上 mq9
5. **mq9 和 A2A 协议的融合** —— Google 推的 Agent 间同步协议，150+ 企业在用，异步这块是空的，正是 mq9 的位置
6. **CrewAI / AutoGen 集成**
7. **技术文章、深度博客**
8. **Hacker News / X / Reddit 推广**
9. **公共 mq9 server + 可视化页面** —— 所有人的 Agent 在上面通信，打开网页就能看到谁在跑、谁在聊
10. **一起讨论、探索 Agent 通信的未来**

最近想先做第 9 和第 4 条。

想象一下：跑 OpenClaw 的人装一个插件，Agent 就接入了一个全球的 Agent 通信网络。打开网页，能看到自己的 Agent 和其他 Agent 在异步通信、领取任务、广播事件。

这是一个以前不存在的东西。

## 为什么只要一个人

不是谦虚，是判断。

AI Coding 时代，人多不一定是优势。多一个人多一份沟通摩擦，多一份协调成本。我不相信人海战术能做好一个小而精的基础设施项目。

而且说实话，除了 mq9 内核需要认真打磨的代码——其实内核也包括——我们几乎所有事情都会在 AI 辅助下完成。文档、博客、推广、集成、SDK、插件、网页，AI 能帮我们做大部分。

所以我要找的不是人力。我要找的是一个真正想做这件事的人。

不是传统意义上的"开发者"。是一个有判断、有好奇心、愿意和 AI 一起推一件事的人。

## 你能得到什么

先说清楚：**没钱。一点没有。**

不是"暂时没钱以后融到资分你"。我们没打算商业化。就是想试试这事能不能成。

所以你能得到的是这些：

**你会得到 AI 在各个领域的实战锻炼。** Coding、思考、研究、写作、推广、协作，每一个方向都是真实的 AI 实战场景。不是玩具项目，是一个正在生长的开源基础设施。

**你会真正深度切入 AI Infra。** 不是看分析文章，不是听分享，是你自己在建造这一层。去了解它，去拥抱它。这个机会不是任何一份工作能给的。

**你会看到一件事从无到有。** 没有 KPI，没有融资压力，纯粹想试试能不能做成。成了是大事，不成也没什么。

**你会和真心想做这件事的人一起走。** RobustMQ 社区已经在了，mq9 是最新的方向。我们不是萍水相逢，是想把一件事做对的同路人。

如果这些对你有吸引力，继续往下。如果你想的是简历镀金、推荐信、大厂跳板，不是这里。

## 我希望你

**真的想做。** 不是三分钟热度，不是朋友圈立 flag，不是"先来看看"。

**有时间。** 不要求全职，不要求每天几小时。但你得有节奏，一周能稳定拿出几段完整时间。不一定长，但要规律。

**有激情。** 对 Agent、对基础设施、对"下一代通信长什么样"有真正的好奇。聊起来会兴奋，看到别人做的东西忍不住研究。

**不懂没关系。** 协议不懂可以学，Rust 不会可以学，LangChain 不熟上手试就行。AI Coding 时代，真正稀缺的不是技能，是那股想搞清楚、想做出来的劲儿。
