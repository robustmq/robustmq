# RobustMQ 里程碑：MQTT 写，一份数据，MQTT、Kafka、NATS、AMQP 消费

今天，RobustMQ 完成了一个关键验证。

## 一次写入，多协议消费
一条消息，通过 MQTT 写入。无需任何桥接、转发、数据复制——MQTT、Kafka、NATS、AMQP 四个协议的消费端，同时收到了同一条消息。

```bash
# 发布
mqttx pub -t 'robustmq.multi.protocol' -h '127.0.0.1' -p 1883 -q 1 \
  -u 'admin' -P 'robustmq' -m 'robustmq multi protocol success'
```

四端同时消费：

- **MQTT**：`mqttx sub -t 'robustmq.multi.protocol'`
- **Kafka**：`kafka-console-consumer.sh --topic robustmq.multi.protocol`
- **NATS**：`nats sub "robustmq.multi.protocol"`
- **AMQP**：`python3 amqp_consumer.py`

这不是协议转换，不是数据桥接，不是 fan-out 复制写入。

底层只有一份数据，存在统一的 Shard 存储层里。四种协议各自从同一份存储中读取——写一次，任意协议消费。

没有任何新增概念。你用的还是原生的 MQTT、Kafka、NATS、AMQP——原生 SDK，原生命令，原生语义。RobustMQ 不要求你学任何新东西，理解成本为零。

这是 RobustMQ 架构的核心主张第一次在运行时得到完整验证：**统一存储，多协议原生支持**。

单个二进制，零外部依赖。

路还很长，但这一步走实了。
