# $mq9.AI.* 协议设计文档

## 概述

`$mq9.AI.*` 是 mq9 为 Agent 异步通信设计的协议。核心解决的问题：**Agent 之间的异步通信，发件方和收件方不需要同时在线。**

mq9 只解决通信问题——消息怎么可靠送达。至于消息里装什么、怎么用、业务语义是什么，mq9 不管。消息内容是 byte 数组，不解析、不校验、不限制。

---

## 基础概念

**mail_address**：通过 `MAILBOX.CREATE` 创建邮箱时返回的通信地址。不绑定 Agent 身份，一个 Agent 可以为不同任务申请不同的 mail_address。用完不管，TTL 自动清理。

私有邮箱的 mail_address 由系统生成，格式为 `mail@{uuid1}.{uuid2}`；指定 prefix 时为 `{prefix}@{uuid1}.{uuid2}`。不可猜测。公开邮箱的 mail_address 由用户自定义，有意义的名字——`task.queue`、`analytics.result`。

**mail_address 不可猜测即安全边界。** 知道 mail_address 就能发消息、能订阅。不知道 mail_address 就无从操作。没有 token，没有 ACL。

**TTL**：邮箱创建时声明，到期自动销毁，消息随之清理。CREATE 幂等，重复创建同一邮箱不报错，TTL 以第一次创建为准。

**priority**：可选。不指定为 normal 默认优先级，指定 urgent 或 critical 可提升处理顺序。同优先级 FIFO，跨优先级高优先处理。存储层保证顺序，消费方无需自行排序。

**msg_id**：每条消息的唯一标识（消息在存储中的 offset），客户端用于去重和删除操作。

**订阅语义**：每次订阅全量推送所有未过期消息，后续新消息实时推送。不区分已读/未读，不追踪消费位点。服务端零消费状态。

**消息流程**：消息到达 → 写存储 → 在线则实时推送，不在线则等待 → 订阅者上线时全量推送所有未过期消息。

---

## 协议总览

| Subject | 方向 | 持久化 | 说明 |
|---------|------|--------|------|
| `$mq9.AI.MAILBOX.CREATE` | REQ/REP | — | 创建邮箱 |
| `$mq9.AI.MAILBOX.MSG.{mail_address}` | REQ/REP | 是 | 发消息（normal 优先级） |
| `$mq9.AI.MAILBOX.MSG.{mail_address}.urgent` | REQ/REP | 是 | 发消息（紧急） |
| `$mq9.AI.MAILBOX.MSG.{mail_address}.critical` | REQ/REP | 是 | 发消息（最高优先级） |
| `$mq9.AI.MAILBOX.MSG.{mail_address}` | SUB | — | 订阅接收消息 |
| `$mq9.AI.MAILBOX.LIST.{mail_address}` | REQ/REP | — | 查看邮箱当前内容 |
| `$mq9.AI.MAILBOX.DELETE.{mail_address}.{msg_id}` | REQ/REP | — | 删除指定消息 |

> 所有命令均采用 request/reply 模式（`nats request`），server 必定返回响应。订阅（SUB）使用 `nats subscribe`，server 主动推送。

---

## 统一响应格式

所有命令共用同一个 JSON 响应结构（`Mq9Reply`），字段按需填充：

```json
{
  "error": "",
  "mail_address": "...",
  "is_new": true,
  "msg_id": 8,
  "messages": [...],
  "deleted": true
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 始终存在。成功时为空字符串，失败时为错误描述 |
| `mail_address` | string? | CREATE 和 LIST 成功时返回 |
| `is_new` | bool? | CREATE 成功时返回，`true` 表示新建，`false` 表示已存在 |
| `msg_id` | u64? | MSG 发送成功时返回，值为消息在存储中的 offset |
| `messages` | array? | LIST 成功时返回，消息元数据列表 |
| `deleted` | bool? | DELETE 成功时返回，值固定为 `true` |

## $mq9.AI.MAILBOX.CREATE

创建邮箱。私有邮箱 mail_address 由系统生成，公开邮箱 mail_address 由用户自定义。

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `ttl` | u64? | 否 | 邮箱存活时间（秒），不填则永不过期 |
| `public` | bool | 否 | 是否为公开邮箱，默认 `false` |
| `name` | string? | 否 | 公开邮箱自定义名称（`public: true` 时有效） |
| `prefix` | string? | 否 | 私有邮箱地址前缀，格式为小写字母、数字和点，生成地址为 `{prefix}@{uuid1}.{uuid2}` |
| `desc` | string | 否 | 邮箱描述 |

### CREATE 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息 |
| `mail_address` | string | 创建成功后的邮箱地址 |
| `is_new` | bool | `true` 表示新建，`false` 表示邮箱已存在（幂等） |

### CREATE 示例

```bash
# 创建私有邮箱
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl": 3600}'
# 响应
{"error":"","mail_address":"mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00","is_new":true}

# 创建带前缀的私有邮箱
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl": 3600, "prefix": "risk.critical"}'
# 响应
{"error":"","mail_address":"risk.critical@d7ms26alko8787uunnvg.d7ms26alko8787uuno01","is_new":true}

# 创建公开邮箱
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl":3600,"public":true,"name":"task.queue","desc":"任务队列"}'
# 响应
{"error":"","mail_address":"task.queue","is_new":true}

# 重复创建（幂等）
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl": 3600, "name": "task.queue", "public": true}'
# 响应
{"error":"","mail_address":"task.queue","is_new":false}
```

## $mq9.AI.MAILBOX.MSG.{mail_address}

消息通道。同一个 mail_address，REQ 发消息，SUB 接收消息。

### 发消息（REQ/REP）

知道 mail_address 即可发送，无需任何授权。

#### MSG 请求参数

| 参数 | 说明 |
|------|------|
| Subject | `$mq9.AI.MAILBOX.MSG.{mail_address}` — normal 优先级（默认）|
| Subject | `$mq9.AI.MAILBOX.MSG.{mail_address}.urgent` — 紧急 |
| Subject | `$mq9.AI.MAILBOX.MSG.{mail_address}.critical` — 最高优先级 |
| Payload | 任意 byte 数组，mq9 不解析 |

#### MSG 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息（如邮箱不存在） |
| `msg_id` | u64 | 消息写入存储后的 offset，客户端可用于去重和删除 |

#### 优先级说明

| 值 | Subject 后缀 | 语义 | 典型场景 |
|----|-------------|------|---------|
| normal | 无后缀 | 默认，大多数消息 | 任务分发、结果返回、状态上报 |
| urgent | `.urgent` | 紧急，优先处理 | 审批请求、重要通知 |
| critical | `.critical` | 最高优先级 | 任务中断、紧急指令 |

同优先级 FIFO，跨优先级 critical > urgent > normal。

#### MSG 示例

```bash
nats request '$mq9.AI.MAILBOX.MSG.mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00' 'hello'
# 响应
{"error":"","msg_id":0}

nats request '$mq9.AI.MAILBOX.MSG.mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00.urgent' 'urgent payload'
# 响应
{"error":"","msg_id":1}

nats request '$mq9.AI.MAILBOX.MSG.mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00.critical' 'abort'
# 响应
{"error":"","msg_id":2}

# 邮箱不存在
nats request '$mq9.AI.MAILBOX.MSG.nonexistent' 'hello'
# 响应
{"error":"mailbox nonexistent does not exist"}
```

### 接收消息（SUB）

```bash
# 订阅所有消息（按优先级推送：critical → urgent → normal）
nats subscribe '$mq9.AI.MAILBOX.MSG.{mail_address}'

# 竞争消费（每条消息只被一个订阅者处理）
nats subscribe '$mq9.AI.MAILBOX.MSG.{mail_address}' --queue workers
```

订阅即全量推送——所有未过期消息立即按优先级顺序推送，后续新消息实时推送。

**禁止的操作**：

```bash
nats subscribe '$mq9.AI.MAILBOX.MSG.*'  # 禁止，broker 拒绝
nats subscribe '$mq9.AI.MAILBOX.MSG.#'  # 禁止，broker 拒绝
```

订阅必须精确到 mail_address，不允许通配符订阅所有邮箱。

## $mq9.AI.MAILBOX.LIST.{mail_address}

查看邮箱当前存储的所有消息元数据，不影响订阅推送，不消费消息。

### LIST 请求参数

| 参数 | 说明 |
|------|------|
| Subject | `$mq9.AI.MAILBOX.LIST.{mail_address}` |
| Payload | 空（`''`） |

### LIST 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息 |
| `mail_address` | string | 邮箱地址 |
| `messages` | array | 消息元数据列表，按优先级排序 |

**`messages` 数组元素字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `msg_id` | u64 | 消息 offset，全局唯一，用于删除操作 |
| `payload` | string | 消息内容（byte 数组的字符串表示） |
| `priority` | string | 优先级：`critical` / `urgent` / `normal` |
| `header` | bytes? | NATS Header（可选） |
| `create_time` | u64 | 消息写入时间戳（Unix 秒） |

### LIST 示例

```bash
nats request '$mq9.AI.MAILBOX.LIST.mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00' ''
# 响应
{
  "error": "",
  "mail_address": "mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00",
  "messages": [
    {"msg_id": 2, "payload": "abort",          "priority": "critical", "header": null, "create_time": 1234567892},
    {"msg_id": 1, "payload": "urgent payload", "priority": "urgent",   "header": null, "create_time": 1234567891},
    {"msg_id": 0, "payload": "hello",          "priority": "normal",   "header": null, "create_time": 1234567890}
  ]
}
```

## $mq9.AI.MAILBOX.DELETE.{mail_address}.{msg_id}

删除邮箱中的指定消息。已处理的消息可以显式清理，保持邮箱干净。

### DELETE 请求参数

| 参数 | 说明 |
|------|------|
| Subject | `$mq9.AI.MAILBOX.DELETE.{mail_address}.{msg_id}` |
| Payload | 空（`''`） |

`msg_id` 为 LIST 或发消息时返回的 offset 值。

### DELETE 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息 |
| `deleted` | bool | 成功时固定为 `true` |

### DELETE 示例

```bash
nats request '$mq9.AI.MAILBOX.DELETE.mail@d7ms26alko8787uunnvg.d7ms26alko8787uuno00.2' ''
# 响应
{"error":"","deleted":true}
```

## 公开邮箱

公开邮箱没有独立的发现机制。mail_address 由用户自定义为有意义的名字——`task.queue`、`analytics.result`——知道名字直接订阅即可：

```bash
nats subscribe '$mq9.AI.MAILBOX.MSG.task.queue'
```

创建时指定 `public: true` 和自定义名字：

```bash
nats request '$mq9.AI.MAILBOX.CREATE' '{"ttl":3600,"public":true,"name":"task.queue","desc":"任务队列"}'
# 响应
{"error":"","mail_address":"task.queue","is_new":true}
```

公开邮箱和私有邮箱的区别只在于 mail_address 是否可预测，通信机制完全一致。

---

## 错误一览

| 场景 | 响应示例 |
|------|---------|
| 邮箱不存在（发消息/订阅/LIST/DELETE） | `{"error":"mailbox xxx does not exist"}` |
| 邮箱已存在（CREATE 幂等，不报错） | `{"error":"","mail_address":"xxx","is_new":false}` |
| msg_id 不存在（DELETE） | `{"error":"message not found"}` |
