# $mq9.AI.* 协议设计文档

## 概述

`$mq9.AI.*` 是 mq9 为 Agent 异步通信设计的协议。三个命令字覆盖所有 Agent 通信场景：创建邮箱/频道、点对点投递、公共广播。

核心解决的问题：**Agent 之间的异步通信，发件方和收件方不需要同时在线。**

mq9 只解决通信问题——消息怎么可靠送达。至于消息里装什么、怎么用、业务语义是什么，mq9 不管。消息内容是 byte 数组，不解析、不校验、不限制。

---

## 基础概念

**mail_id**：通过 `MAILBOX.CREATE` 创建 INBOX 时返回的全局唯一通信地址。不绑定 Agent 身份，一个 Agent 可以为不同任务申请不同的 mail_id。用完不管，TTL 自动清理。

**邮箱类型**：
- `standard`（默认）：消息累积，按优先级存储。
- `latest`：只保留最新一条，新消息覆盖旧消息。

**TTL**：INBOX 和 BROADCAST 创建时声明，到期自动销毁，消息随之清理。无显式删除操作。TTL 以第一次 CREATE 为准，重复 CREATE 不覆盖。

**priority**：INBOX 支持三个优先级：urgent、normal、notify。同优先级 FIFO，跨优先级高优先处理。

**token**：创建 INBOX 时返回。任何人知道 mail_id 即可发消息，token 仅用于邮箱管理操作。BROADCAST 无 token。

**msg_id**：每条消息的唯一标识，客户端用于去重。

**订阅语义**：INBOX 和 BROADCAST 行为一致——每次订阅全量推送所有未过期消息。不区分已读/未读，不追踪消费位点。服务端零消费状态。

**消息流程**：消息到达 → 写存储 → 在线则实时推送，不在线则等待 → 订阅者下次订阅时全量推送所有未过期消息。

**CREATE 幂等性**：重复创建同一个 INBOX 或 BROADCAST 不报错，静默返回成功。TTL 以第一次创建为准。

**无状态查询**：mq9 不提供"邮箱是否存在""频道是否存在"的查询接口。CREATE 永远返回成功。

## 命令字

| 命令字 | Subject | 操作 | 说明 |
|--------|---------|------|------|
| `MAILBOX.CREATE` | `$mq9.AI.MAILBOX.CREATE` | REQ | 创建邮箱或广播频道。INBOX 返回 mail_id + token，BROADCAST 返回成功 |
| `INBOX` | `$mq9.AI.INBOX.{mail_id}.{priority}` | PUB | 点对点发送消息，持久化，支持离线接收 |
| `INBOX` | `$mq9.AI.INBOX.{mail_id}.*` | SUB | 订阅邮箱，全量接收所有未过期消息 |
| `BROADCAST` | `$mq9.AI.BROADCAST.{domain}.{event}` | PUB | 公共广播，持久化，任何人可发 |
| `BROADCAST` | `$mq9.AI.BROADCAST.{domain}.{event}` | SUB | 订阅广播频道，全量接收所有未过期消息，支持通配符和 queue group |
| `BROADCAST` | `$mq9.AI.BROADCAST.system.*` | SUB | 订阅系统公告板（系统内置，TTL 永不过期） |

---

## INBOX 与 BROADCAST 的关系

两者底层行为一致：持久化、TTL 自动清理、订阅全量推送、CREATE 幂等。区别是权限模型。

| | INBOX（私人邮箱） | BROADCAST（公共频道） |
|---|---|---|
| CREATE 返回 | mail_id + token | 成功（无 mail_id，无 token） |
| 谁能发 | 任何人（知道 mail_id） | 任何人 |
| 谁能收 | 只有邮箱主人 | 任何人 |
| 身份验证 | token 管理邮箱 | 无 |
| 优先级 | urgent / normal / notify | 无 |
| 用途 | 私人通信 | 公共论坛 / 公告板 |

## $mq9.AI.MAILBOX.CREATE

统一的创建入口。通过请求中的 subject 前缀区分创建 INBOX 还是 BROADCAST。

### 创建私人邮箱（INBOX）

```bash
REQ $mq9.AI.MAILBOX.CREATE
{
  "type": "standard",       # standard | latest
  "ttl": 3600,              # 邮箱生存时间（秒）
  "subject": "$mq9.AI.INBOX"
}

# 响应
{
  "mail_id": "m-uuid-001",
  "token": "tok-xxx",
  "inbox": "$mq9.AI.INBOX.m-uuid-001"
}
```

### 创建广播频道（BROADCAST）

```bash
REQ $mq9.AI.MAILBOX.CREATE
{
  "type": "standard",       # standard | latest
  "ttl": 3600,              # 频道生存时间（秒）
  "subject": "$mq9.AI.BROADCAST.pipeline.complete"
}

# 响应
{
  "subject": "$mq9.AI.BROADCAST.pipeline.complete"
}
```

服务端识别 `$mq9.AI.BROADCAST.` 前缀，不生成 mail_id，不生成 token。频道创建后任何人可发可订。

CREATE 是幂等的。频道已存在则静默返回成功，TTL 以第一次创建为准，后续不覆盖。

## $mq9.AI.INBOX.{mail_id}.{priority}

点对点消息投递。知道 mail_id 即可发送，无需 token。

```bash
PUB $mq9.AI.INBOX.{mail_id}.{priority}
{
  "msg_id": "msg-uuid-001",
  "from": "m-uuid-002",
  "type": "task_result",
  "correlation_id": "req-uuid-001",
  "reply_to": "$mq9.AI.INBOX.m-uuid-002.normal",
  "deadline": 1234567890,
  "payload": { ... },
  "ts": 1234567890
}
```

订阅：

```bash
SUB $mq9.AI.INBOX.{mail_id}.*        # 所有优先级
SUB $mq9.AI.INBOX.{mail_id}.urgent   # 仅紧急
```

**priority**

| 值 | 说明 | 持久化 | 建议 TTL |
|----|------|--------|---------|
| urgent | 紧急，优先处理 | 是 | 86400s |
| normal | 普通，顺序处理 | 是 | 3600s |
| notify | 通知，后台处理 | 否 | 短 |

消息随邮箱一起过期。邮箱 TTL 到期，其中所有消息一起销毁。

## $mq9.AI.BROADCAST.{domain}.{event}

公共广播。需要先 CREATE 创建频道，之后任何人可发，任何人可订阅。消息持久化，离线订阅者上线后全量收到未过期消息。

```bash
PUB $mq9.AI.BROADCAST.{domain}.{event}
{
  "msg_id": "msg-uuid-002",
  "from": "m-uuid-004",
  "type": "event_type",
  "severity": "high",
  "reply_to": "$mq9.AI.INBOX.m-uuid-004.normal",
  "payload": { ... },
  "ts": 1234567890
}
```

订阅：

```bash
SUB $mq9.AI.BROADCAST.system.*              # 单域所有事件
SUB $mq9.AI.BROADCAST.*.anomaly             # 跨域特定事件
SUB $mq9.AI.BROADCAST.#                     # 所有广播
SUB $mq9.AI.BROADCAST.task.available  queue_group=task.workers  # 竞争消费
```

### 系统公告板

`$mq9.AI.BROADCAST.system.*`，系统启动时自动创建，TTL 永不过期，不可删除。行为跟用户创建的广播频道一致。

| Subject | 用途 |
|---------|------|
| `$mq9.AI.BROADCAST.system.capability` | 能力公告 |
| `$mq9.AI.BROADCAST.system.status` | 状态公告 |
| `$mq9.AI.BROADCAST.system.channel` | 频道公告 |

### domain 建议命名

| domain | 说明 |
|--------|------|
| system | 系统公告板，系统内置 |
| task | 任务相关 |
| data | 数据变更事件 |
| {自定义} | 业务域自定义 |

---

## 协议总览

| Subject | 方向 | 持久化 | 说明 |
|---------|------|--------|------|
| `$mq9.AI.MAILBOX.CREATE` | REQ/RESP | — | 创建邮箱或广播频道 |
| `$mq9.AI.INBOX.{id}.urgent` | PUB/SUB | 是 | 紧急消息 |
| `$mq9.AI.INBOX.{id}.normal` | PUB/SUB | 是 | 普通消息 |
| `$mq9.AI.INBOX.{id}.notify` | PUB/SUB | 否 | 通知 |
| `$mq9.AI.BROADCAST.system.capability` | PUB/SUB | 是 | 系统公告板：能力 |
| `$mq9.AI.BROADCAST.system.status` | PUB/SUB | 是 | 系统公告板：状态 |
| `$mq9.AI.BROADCAST.system.channel` | PUB/SUB | 是 | 系统公告板：频道 |
| `$mq9.AI.BROADCAST.{domain}.{event}` | PUB/SUB | 是 | 用户自定义广播 |
